# VRM / MMD 骨骼适配落地计划

## Summary

下一阶段目标是把目标骨架适配从当前 `Generic / ARP / MMD` 的初步分发，推进到可维护的 `VRM / ARP / MMD` 多目标骨架工作流。

本计划只处理目标骨架适配、校准和映射落地，不重做接收协议、不重做录制架构、不把预览和录制拆成两条运行时分支。

核心约束：

- 原始 VRM 预览骨架继续用于查看 VMC 原始输入。
- 映射后的目标角色预览与录制必须共用同一条目标求值链路。
- VRM / 通用骨架目标直接复用当前 VRM 预览骨架能力，不重新做一套运行时。
- MMD 目标骨架需要从“占位入口”推进到可检测、可匹配、可校准、可录制的模块。

## 当前状态

### 已有能力

- `VMC_TEST_DUMMY` VRM 预览骨架可创建、重建、实时驱动。
- 预览骨架可按目标骨架做基础长度校准。
- 目标骨架类型切换已存在：`Generic / ARP / MMD`。
- `mapping/target_rig.py` 已承接目标类型分发。
- `mapping/mmd.py` 已有 MMD 标准名表、检测、自动填充和校验入口。
- `presets/bone_maps/mmd_humanoid.json` 已存在。
- 录制与目标角色预览已开始共用 `TargetEvaluatedSample`。

### 当前不足

- VRM / 通用骨架与 MMD 的校准逻辑还没有分支，当前目标骨架长度校准更偏通用处理。
- VRM 目标骨架适配没有明确成一个“一键匹配 + 预设 + 校验”的目标类型工作流。
- MMD 骨骼目标当前仍像通用骨架占位，缺少对真实 MMD 骨架的检测报告、轴向修正和角色级验证。
- MMD 根运动、上半身、下半身、手脚、手指和眼睛的职责划分仍需要明确。
- UI 仍有 ARP 历史痕迹，需要继续拆出目标骨架类型对应的辅助区。

## 设计目标

### 1. 校准功能按目标类型分支

预览骨架的“按目标骨架校准”改为按当前目标骨架类型选择策略：

- `VRM / Generic`
  - 使用 VRM humanoid 骨骼语义。
  - 以髋部到肩高为整体比例基准。
  - 只调整四肢、手掌和手指必要骨骼长度。
  - 保持标准 VRM T Pose 形态，不破坏父子链跟随。
- `MMD`
  - 使用 MMD 标准骨骼语义和日文骨名。
  - 单独处理 `センター / 下半身 / 上半身 / 上半身2` 的高度与层级差。
  - 单独处理 MMD 常见 A Pose / T Pose 差异。
  - 手臂、手掌、手指长度按 MMD 目标骨实际链长校准。
  - 脚、つま先、IK 辅助骨不混入第一阶段长度校准。
- `ARP`
  - 保持现有 ARP 校准路径，后续只做生产角色调优。

### 2. VRM 目标骨架复用现有链路

VRM 骨骼目标即当前“通用骨架”主线，不新建独立运行时。

落地内容：

- 新增或整理内建预设：`vrm_humanoid.json`。
- UI 中把 `Generic` 对用户显示为 `VRM / 通用骨架`。
- 增加“一键匹配 VRM 骨架”操作。
- 增加“校验 VRM 映射”报告：
  - 必需骨覆盖率。
  - 可选眼睛和手指覆盖率。
  - 缺失骨列表。
  - 已手动填写项不覆盖。
- 目标预览和录制继续使用当前统一目标求值链路。

明确不做：

- 不为 VRM 目标重新做第二套预览骨架。
- 不在录制阶段直接读取 Blender 当前对象姿态作为真值。
- 不把 VRM 目标路径和 ARP / MMD 的特殊逻辑混在一起。

### 3. MMD 骨骼模块真正落地

MMD 模块需要从“名表 + 占位入口”推进到目标骨架适配器。

第一阶段骨骼范围：

- 根与躯干：
  - `センター`
  - `下半身`
  - `上半身`
  - `上半身2`
  - `首`
  - `頭`
- 手臂：
  - `左肩 / 右肩`
  - `左腕 / 右腕`
  - `左ひじ / 右ひじ`
  - `左手首 / 右手首`
- 腿脚：
  - `左足 / 右足`
  - `左ひざ / 右ひざ`
  - `左足首 / 右足首`
  - `左つま先 / 右つま先`
- 手指：
  - `親指 / 人指 / 中指 / 薬指 / 小指`
  - 各手指 `0 / 1 / 2 / 3` 或目标骨架实际分节命名。
- 可选：
  - `左目 / 右目`
  - 捩骨、IK 骨、補助骨先不作为第一阶段默认驱动目标。

MMD 运行时策略：

- 根位移默认写入 `センター`。
- 躯干旋转按 `下半身 / 上半身 / 上半身2` 分配，不直接把 VRM Hips 全量套到单骨。
- 手臂和腿脚使用目标骨 rest 轴重映射，不直接复制 VRM 骨骼旋转数值。
- 手指优先走局部 basis delta，避免左右手、拇指和分节轴向反转。
- 第一阶段不驱动 IK 骨，保留 FK 曲线，方便后期手动修帧。

## 实现阶段

### Phase 1：读取当前工程 MMD 骨架并形成测试报告

目标：

- 用 Blender MCP 读取当前工程内用于测试的 MMD 角色骨架。
- 输出骨架对象名、骨骼层级、骨骼 head/tail、roll、父子关系和可识别 MMD 标准骨覆盖率。

产物：

- 当前角色 MMD 骨架差异报告。
- `mapping/mmd.py` 名表修正清单。
- 校准策略需要额外处理的骨骼列表。

验收：

- 能明确当前工程 MMD 骨架是否符合标准 MMD 命名。
- 能明确缺失项、别名项和不应驱动项。

### Phase 2：VRM / 通用骨架一键匹配

目标：

- 把当前通用骨架路径明确产品化为 `VRM / 通用骨架`。

实现：

- 新增内建 `vrm_humanoid.json` 或整理现有通用预设。
- 新增 `检查 VRM 骨架`、`一键匹配 VRM 骨架`、`校验 VRM 映射`。
- UI 中将该目标类型和 ARP / MMD 并列展示。

验收：

- VRM humanoid 目标骨架可以一键填充。
- 录制和目标预览仍复用统一目标求值链路。
- 不影响 ARP 和 MMD 目标类型。

### Phase 3：MMD 检测与自动匹配

目标：

- 让 MMD 辅助区能给出真实、可执行的检测结果。

实现：

- 完善 `mapping/mmd.py` 的必需骨、可选骨、手指、眼睛、别名表。
- 自动填充只写入唯一可确定骨骼。
- 不覆盖用户已有手动映射。
- 报告中区分：
  - 必需缺失。
  - 可选缺失。
  - 手指缺失。
  - 歧义项。
  - 不参与驱动的 IK / 辅助骨。

验收：

- 当前工程 MMD 骨架可被识别。
- 自动填充报告能说明写入项和跳过项。
- 非 MMD 骨架不会被模糊兜底识别为 MMD。

### Phase 4：MMD 校准分支

目标：

- 预览 VRM 骨架按 MMD 目标角色比例校准，同时保持 VRM T Pose 结构稳定。

实现：

- 抽出校准策略分发：
  - `calibrate_preview_to_vrm_target`
  - `calibrate_preview_to_mmd_target`
  - `calibrate_preview_to_arp_target`
- MMD 策略以髋部到肩高作为整体比例基准。
- 四肢、手掌、手指按目标骨链长局部修正。
- 校准结束后应用位置变换，确保骨盆或脚底基准不下沉。

验收：

- 校准后 VRM 预览骨架不破坏 T Pose。
- 手和手指跟随父级骨骼长度变化。
- 校准后对象不会整体下沉。
- MMD 与 VRM 目标使用不同策略，不互相污染。

### Phase 5：MMD 旋转与录制验证

目标：

- MMD 目标角色在实时预览和录制回放中表现一致。

实现：

- 按起始 pose delta 和目标骨 rest 轴做旋转重映射。
- 单独校验肩膀、手臂、手腕、拇指、脚、脚趾。
- 根运动只进入 `センター` 或最终确认的 MMD 根运动骨，不移动无关对象。
- 录制继续走统一目标求值和停止后烘焙。

验收：

- 实时预览和录制回放姿态一致。
- 手臂不出现 90 度轴向偏差。
- 拇指不出现内弯变外翻。
- 脚部旋转不直接继承 VRM 脚骨错误轴向。
- 录制 Action 后续可手动编辑 FK 曲线。

## UI 调整

骨骼映射面板按目标类型显示辅助区：

- `VRM / 通用骨架`
  - 检查 VRM 骨架
  - 一键匹配 VRM 骨架
  - 校验 VRM 映射
- `Auto Rig Pro`
  - 检查 ARP 骨架
  - 应用标准 ARP 映射
  - 校验当前映射
- `MMD`
  - 检查 MMD 骨架
  - 一键匹配 MMD 骨架
  - 校验 MMD 映射

中间层预览面板的“按目标校准长度”需要显示当前采用的校准策略，例如：

- `当前校准策略：VRM / 通用骨架`
- `当前校准策略：MMD`
- `当前校准策略：Auto Rig Pro`

## 测试计划

静态检查：

- `python3 -m py_compile __init__.py core/*.py runtime/*.py mapping/*.py preview/*.py ui/*.py`
- `git diff --check`

Blender 内验证：

- MCP 读取当前工程 MMD 骨架并保存差异报告。
- VRM / 通用骨架一键匹配后，目标预览和录制仍正常。
- MMD 一键匹配后，检测报告能准确区分必需骨和可选骨。
- MMD 校准后预览骨架不下沉、不破坏 T Pose、手指链跟随父级长度。
- MMD 实时预览和录制回放结果一致。
- ARP 路径不因 VRM / MMD 改动产生回归。

## 当前优先级

1. 用 MCP 读取当前工程 MMD 骨架，形成开发测试基准。
2. 抽出预览骨架校准策略分发，先把 VRM / MMD 分支边界建立起来。
3. 将 VRM / 通用骨架路径产品化为预设 + 一键匹配 + 校验。
4. 补齐 MMD 检测、自动匹配和校验报告。
5. 做 MMD 旋转轴向、手指、脚部和录制回放验证。
