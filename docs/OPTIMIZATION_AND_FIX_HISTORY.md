# 优化与修复记录

本文档收纳 README 中移出的优化、修复和性能收敛记录。README 只保留当前能力与入口说明。

## 当前结论

- ARP 映射模块已进入可用阶段，后续主要是生产角色调优。
- 录制链路已从“实时写 Action”改为“录制期缓存、停止后烘焙”，并开始用原始帧时间戳重建目标结果。
- 目标角色实时预览与录制已经共用统一目标求值结果，不应再拆成两条互相漂移的链路。
- MMD 骨骼目标适配仍处于落地阶段，现有实现更多是通用骨架能力的占位和初步分发。

## 录制链路修复

- 录制入口已拆成独立面板，支持显式开始帧、结束帧、自动停止、清空范围关键帧和起始姿态过渡。
- 开始录制前会跳到设定开始帧，并自动启动或继续接收。
- 手动停止录制和到达结束帧后的自动停止会复用统一停止接收链路。
- 清空范围关键帧前后会回到录制开始帧，方便角色恢复初始姿态。
- 录制会复用当前 Action；没有 Action 时才创建，不再每次录制新建 Action。
- 录制期间会临时从目标对象解绑 Action，避免 Action 评估覆盖实时目标姿态。
- Shape Key 录制写入 Shape Keys 数据块对应的 `KEY` action slot。
- Blender 5.1 layered action slot 重新绑定已纳入停止录制流程。
- ARP 录制会写入手脚 IK/FK 开关关键帧，保证 FK 控制骨曲线能参与回放。
- 起始姿态过渡源改为接收启动时保存的目标初始姿态。
- 起始姿态过渡终点改为过渡结束帧对应的有效动捕姿态。

## 录制性能修复

- 录制阶段不再实时扫描、插入和更新 `keyframe_points`。
- 录制期间缓存原始帧或目标求值结果，停止录制后统一烘焙 Action。
- 停止录制后的保存流程拆成“准备保存 -> 重建采样 -> 写入动作曲线”，并接入 Blender 状态栏进度条。
- 录制帧号按真实时间和工程 FPS 换算，修复录制动作快放问题。
- 跨帧 tick 会按相邻采样补齐中间帧。
- 位置和 Shape Key 使用线性补帧；骨骼旋转使用 quaternion slerp 后写回目标旋转模式。
- 烘焙后的关键帧默认写为 `LINEAR`，避免默认 `BEZIER` 造成曲线段不均匀。

## 实时预览性能修复

- UDP 收包和 OSC 解码在后台线程执行，Blender RNA 写入保留在主线程。
- 接收线程维护 canonical 骨骼与表情缓冲，主线程不再每 tick 做整表规范化。
- 主线程只消费最新实时帧用于预览；录制仍保留原始帧序列用于离线烘焙。
- 运行时按 `dirty_bone_names / dirty_vmc_blend_names / dirty_arkit_blend_names` 增量求值。
- 身体、VMC 表情、ARKit 表情的脏标记已拆分，避免 ARKit-only 或普通表情输入误触发整套骨骼求值。
- VMC `Look*` 眼球表情与普通表情已拆分，只有眼球相关输入才触发眼球骨路径。
- 预览层、目标层和 Shape Key 写入会缓存上次写入值，只有数值变化时才写 Blender RNA。
- `apply_timer()` 只在真实写入或 UI 状态需要刷新时触发 `VIEW_3D` redraw。
- `VIEW_3D` area 列表会缓存，避免每次 redraw 扫描所有窗口区域。
- 空闲状态不再按 10Hz 持续刷新 3D 视图，仅在状态文案需要变化时低频刷新。

## 目标求值链路收敛

- 已新增 `runtime/target_runtime.py` 和 `TargetEvaluatedSample`。
- 目标角色实时预览与录制采样开始共用统一目标求值结果。
- 当前目标运行时入口覆盖 `Generic / ARP / MMD` 和目标 Shape Key。
- 接收层维护 `RawMotionFrame` 队列，目标预览和录制可按原始帧时间戳消费。
- ARP 目标层当前直接按原始 `bones` 求值，不再依赖预览 VRM 骨架当前 pose。
- `mapping/target_rig.py` 承接目标骨架类型选择、默认映射应用、报告清理和运行时分发。
- 骨骼映射面板已加入目标骨架类型切换和 ARP / MMD 辅助区。

## ARP 修复记录

- 标准 ARP 控制骨检测、自动填充和校验可用。
- 内建 `arp_fk_humanoid.json` 预设可用。
- 接收时自动切换四肢 FK，停止后恢复。
- VRM T Pose 与 ARP A Pose 姿态差已经做基础校准。
- 手指局部 basis delta 驱动已接入。
- 拇指 rest 轴转换已修复内弯变外翻问题。
- 脚和脚趾使用相对起始姿态的 delta 旋转，避免直接套源脚部旋转。
- ARP 根运动只写 `c_traj.location`，保持 `rig` 对象和 `c_pos` 默认不动。
- 近期重点仍是肩膀、上臂和局部轴向在真实角色上的生产调优。

## MMD 当前状态

- MMD 形态键映射已具备基础可用状态。
- 已有 VMC 表情映射：`presets/vmc_blend_maps/arona_vmc.json`。
- 已有 ARKit 表情映射：`presets/arkit_blend_maps/arona_arkit.json`。
- MMD 骨骼目标当前已有目标类型入口、默认名表、预设入口和基础运行时分发。
- MMD 骨骼目标仍需要按真实 MMD 骨架落地检测、校准、轴向修正、手脚和手指细节。

## 保留问题

- 长时间真实 VMC / ARKit 接收流下的录制回放仍需系统验证。
- ARP FK 腿部动作转 IK 控制器工具待开发，用于后期脚部 IK 手动修帧。
- 映射 UI 仍需继续收敛，避免 ARP / MMD / VRM 目标类型互相污染。
- `VRM -> MMD` 需要独立开发计划推动，不应继续停留在通用骨架占位状态。
