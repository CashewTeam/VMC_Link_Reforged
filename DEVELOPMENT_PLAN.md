# Development Plan

## 项目阶段判断

VMC Link 当前已经完成基础链路建设，进入“生产适配前调试与模块拆分”阶段。

当前不再把重点放在新增更多输入源，而是围绕以下链路继续收敛：

```text
VMC / RhyLive ARKit 输入
  -> VRM / ARKit 中间层实时预览
  -> 可切换目标映射模块
  -> 目标骨架 / 目标面部实时驱动
  -> Blender 动作与形态键录制
```

## 当前开发状态

### 已完成并验证

#### 1. 基础接收与预览链路

- VMC UDP 接收可用。
- RhyLive ARKit `/Face` 接收可用。
- OSC 收包与解码已放到后台线程。
- Blender 对象、骨骼和 Shape Key 写入保留在主线程。
- VRM 预览骨架可实时驱动。
- ARKit 调试面部可实时驱动。
- 数据预览已移到底部并默认折叠。

#### 2. 中间层与目标层解耦

- 预览层对象：
  - `vmc_link_preview_armature`
  - `vmc_link_preview_face_object`
- 目标层对象：
  - `vmc_link_armature`
  - `vmc_link_face_object`
- 预览层只看原始数据，不读取用户映射。
- 目标层按映射驱动，并受 `实时驱动目标对象` 开关控制。
- 停止接收时可以恢复接收开始前的骨架与面部状态。

#### 3. VRM 中间层

- `VMC_TEST_DUMMY` 预览骨架创建 / 重建可用。
- 创建和重建限定在 `VMC_Link` 集合内，避免误删场景其他骨架。
- 支持按目标骨架执行长度校准。
- 当前校准逻辑已覆盖：
  - 躯干整体比例。
  - 四肢主链长度。
  - 手掌长度。
  - 手指分节长度。
  - 校准后对象位置回归和骨盆点对齐。

#### 4. ARKit 调试面部

- 从 `assets/Arkitface.blend` 只追加 `ArkitFace` 集合。
- 导入对象放入插件专用集合 `VMC_Link`。
- 不覆盖目标面部槽位。

#### 5. Auto Rig Pro 骨骼映射

ARP 映射模块当前已验证为可用阶段，仅待生产环境继续具体调优。

已完成能力：

- 标准 ARP 控制骨检测。
- 标准 ARP 映射自动填充。
- 当前映射校验。
- 内建 `arp_fk_humanoid.json` 预设。
- 接收时自动切换四肢 FK，停止后恢复。
- VRM T Pose 与 ARP A Pose 姿态差校准。
- 手指局部 basis delta 驱动。
- 拇指 rest 轴转换，避免内弯变外翻。
- 脚和脚趾使用相对起始姿态的 delta 旋转。
- ARP 根运动只写 `c_traj.location`，保持 `rig` 对象和 `c_pos` 不动。
- 运行时缓存骨引用、映射和校准结果。
- 跳过未变化骨骼，减少不必要写入。

当前结论：

- ARP 路径已经从“实验实现”进入“可用并待生产调优”。
- 后续不应继续把 ARP 逻辑堆进通用映射 UI，而应拆成可切换目标骨架类型的独立模块。

#### 6. MMD 形态键映射

当前已新增并验证：

- VMC -> MMD Shape Key 映射表。
- ARKit 52 Key -> MMD Shape Key 映射表。
- 两套映射在实时预览状态下功能可用。

现有预设：

- `presets/vmc_blend_maps/arona_vmc.json`
- `presets/arkit_blend_maps/arona_arkit.json`

方向判断：

- `VMC -> 形态键` 继续保留为兼容 / 备选方案。
- 后续面部主线仍以 `ARKit -> MMD` 为主。

### 当前主要问题

#### 1. 录制功能不可用

当前 UI 有录制入口，但实际还不能稳定生成可用的 Blender 动作结果。

需要重新按生产目标定义录制功能：

- 实时骨骼动作可以写入 Action。
- 形态键变化可以写入 Shape Key 动画。
- 录制过程不破坏实时预览。
- 暂停 / 继续 / 停止语义清晰。
- 停止接收后的恢复逻辑不应误删或覆盖录制结果。
- 录制出的结果可以在 Blender 时间轴回放。

#### 2. 映射模块仍未完全解耦

当前骨骼映射 UI 已经被 ARP 辅助区占据，后续加入 MMD 骨骼映射时会变得混乱。

需要新增“目标骨架类型”概念：

- 通用骨架
- Auto Rig Pro
- MMD

目标骨架类型应影响：

- 可见的辅助 UI。
- 自动检测逻辑。
- 自动填充逻辑。
- 校验报告。
- 内建预设入口。
- 运行时映射策略。

#### 3. 缺少 VRM -> MMD 骨骼映射模块

当前已有 MMD 形态键映射，但骨骼侧还没有 MMD 标准骨骼模块。

需要新增：

- MMD 标准骨骼检测。
- VRM -> MMD 默认骨骼映射。
- MMD 映射自动填充。
- MMD 映射校验。
- 内建 MMD 骨骼预设。
- 与 ARP 并列的 UI 辅助区。

#### 4. 生产环境验证不足

当前多数验证来自开发场景和单个角色。

后续需要建立生产验证清单：

- 不同 ARP 角色。
- 至少一套标准 MMD 角色。
- 长时间实时预览。
- 录制回放。
- 表情和骨骼同时录制。
- 固定中心开关切换。
- 暂停 / 继续 / 停止恢复。
- 预设保存、加载、重建缓存。

## 下一阶段路线

## Phase 1：动作与形态键录制修复

状态：`下一步优先`

目标：

- 把当前不可用的录制功能修到可实际用于 Blender 工程。

范围：

- 骨骼关键帧录制。
- Shape Key 关键帧录制。
- 录制状态管理。
- 停止接收与停止录制的边界。
- 录制结果回放验证。

建议实现顺序：

1. 梳理当前录制代码路径，明确哪些对象和骨骼会被 keyframe。
2. 拆分“实时驱动状态恢复”和“录制结果保留”。
3. 为预览层和目标层分别定义是否录制。
4. 优先录制目标层骨架和目标面部。
5. 补 Shape Key 录制，确保 VMC / ARKit 两种来源都能写入动画。
6. 在 Blender 内验证录制结果可回放。

验收标准：

- 开始录制后，目标骨架动作被写入 Action。
- 目标面部 Shape Key 被写入关键帧。
- 停止录制后时间轴可回放。
- 停止接收会恢复实时驱动前状态，但不破坏已录制 Action。
- 录制过程中视口仍能保持可接受帧率。

## Phase 2：映射模块解耦与目标骨架类型切换

状态：`待开始`

目标：

- 不再让 ARP 辅助逻辑直接耦合到通用骨骼映射面板。
- 为 MMD 骨骼映射接入预留清晰入口。

建议新增 Scene 属性：

- `vmc_link_target_rig_type`
  - `GENERIC`
  - `ARP`
  - `MMD`

建议模块结构：

```text
mapping/
  mapper.py          # 通用映射与缓存
  preset_store.py    # JSON 预设
  arp.py             # ARP 目标骨架模块
  mmd.py             # MMD 目标骨架模块
```

UI 调整：

- 骨骼映射面板顶部增加“目标骨架类型”菜单。
- 选择 `Auto Rig Pro` 时显示 ARP 检测、自动填充、校验。
- 选择 `MMD` 时显示 MMD 检测、自动填充、校验。
- 选择 `通用骨架` 时只显示通用映射和预设。

运行时约束：

- `runtime/driver.py` 不应直接包含大量 ARP / MMD UI 或检测逻辑。
- 专用模块负责生成映射和校验。
- 运行时只消费最终映射结果和必要的目标类型策略。

验收标准：

- 切换目标骨架类型不会破坏现有 ARP 实时驱动。
- ARP 辅助区只在 ARP 类型下显示。
- MMD 类型可以独立显示自己的检测与填充入口。
- 现有 JSON 预设仍能加载、保存和刷新。

## Phase 3：VRM -> MMD 骨骼映射模块

状态：`待开始`

目标：

- 新增 MMD 标准骨骼目标适配能力。

建议范围：

- 定义 MMD 标准骨骼名表。
- 定义 VRM 中间层骨骼到 MMD 骨骼的默认映射。
- 支持左右骨、手指、眼睛、上半身链路。
- 提供 MMD 骨架检测。
- 提供自动填充 MMD 映射。
- 提供 MMD 映射校验。
- 新增内建预设，例如：
  - `presets/bone_maps/mmd_humanoid.json`

需要重点验证：

- MMD 骨架坐标轴与 VRM 中间层轴向差异。
- 手指旋转方向。
- 脚和脚趾旋转。
- 根骨 / センター / 下半身 / 上半身 的职责划分。
- 是否需要类似 ARP 的目标专用根运动策略。

验收标准：

- 标准 MMD 骨架可被识别。
- 自动填充不会覆盖用户已有手动映射。
- 校验报告能区分必需骨、可选骨、手指、眼睛。
- 实时驱动时不会影响 ARP 路径。
- 可保存和加载 MMD 骨骼预设。

## Phase 4：生产环境验证与优化

状态：`持续进行`

目标：

- 用真实角色和真实动捕数据验证插件可用性。

验证清单：

- ARP 角色长时间实时预览。
- MMD 角色长时间实时预览。
- 骨骼和形态键同时驱动。
- 骨骼和形态键同时录制。
- 暂停、继续、停止、恢复。
- 固定中心开启 / 关闭。
- 预设保存、加载、刷新。
- 数据预览展开和折叠状态下的性能。

优化方向：

- 目标层独立驱动频率上限。
- Shape Key 写入节流。
- 录制时的关键帧降采样。
- 更清晰的错误报告和缺失项定位。
- 面向生产角色的快速验证按钮。

## 当前优先级

1. 修复录制功能，让动作和形态键可稳定写入 Blender 工程。
2. 拆分映射模块，加入目标骨架类型切换菜单。
3. 接入 VRM -> MMD 骨骼映射模块。
4. 做生产环境验证，基于真实问题继续 debug 和性能优化。

## 当前边界

- 不新增新的输入协议，除非当前 VMC / ARKit 链路已经完成生产验证。
- 不把 ARP 和 MMD 的特殊逻辑混进通用映射路径。
- 不在后台线程写 Blender RNA。
- 不引入会掩盖问题的宽松兜底映射。
- VMC 表情映射保留，但后续面部主线是 ARKit -> MMD。
