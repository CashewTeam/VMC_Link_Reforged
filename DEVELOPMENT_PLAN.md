# Development Plan

## 项目阶段判断

VMC Link 当前已经完成基础链路建设，进入“生产适配前调试与模块拆分”阶段。

当前不再把重点放在新增更多输入源，而是围绕以下链路继续收敛：

```text
VMC / RhyLive ARKit 输入
  -> VRM / ARKit 中间层实时预览
  -> 可切换目标映射模块
  -> 目标骨架 / 目标面部实时驱动
  -> Blender 动作与形态键录制
```

## 当前开发状态

- RhyLive ARKit `/Face` 接收可用。
- OSC 收包与解码已放到后台线程。

- VRM 预览骨架可实时驱动。
- ARKit 调试面部可实时驱动。
- 数据预览已移到底部并默认折叠。

#### 2. 中间层与目标层解耦

- 预览层对象：
  - `vmc_link_preview_armature`
  - `vmc_link_preview_face_object`

- 标准 ARP 映射自动填充。
- 当前映射校验。
- 内建 `arp_fk_humanoid.json` 预设。
- 接收时自动切换四肢 FK，停止后恢复。

### 当前主要问题

#### 1. 录制功能已进入可用阶段，待真实接收流与生产场景验证

当前录制链路已经完成可用性修复，能够用于 Blender 工程内的骨骼与形态键录制；后续重点转为真实接收流长时间验证、生产场景回放验证和后处理工具补充。

相关架构重构方案见：

- `RECORDING_REFACTOR_PLAN.md`

已完成：

- 开始录制时显式准备目标骨架 Action 和目标面部 Shape Key Action。
- 录制入口已提升为独立录制面板，主面板不再混放录制按钮。
- 支持显式录制开始帧 / 结束帧，并在达到结束帧后自动停止。
- 支持按当前录制目标通道清空指定范围内关键帧。
- 点击开始录制前会先把时间线切到设定的录制开始帧。
- 点击开始录制会自动启动接收；如果当前是暂停状态，则自动继续接收。
- 手动停止录制和达到结束帧后的自动停止，都会复用统一的“停止接收”链路。
- 清空指定范围关键帧前会先回到录制开始帧；清空后时间线保持在开始帧，方便角色回到初始姿态。
- 支持可配置的起始姿态过渡：从接收会话启动时保存的目标初始姿态过渡到过渡结束帧对应的有效动捕姿态。
- 录制会复用当前 Action；没有 Action 时才创建。
- 录制期间会临时从目标对象解绑 Action，只缓存目标层结果，停止录制时统一烘焙 fcurve，避免 Action 评估把 ARP 控制骨覆盖回 A Pose。
- 录制会按接收开始前的骨骼 `rotation_mode` 写入旋转曲线，避免运行时 quaternion 模式与回放时 XYZ 模式不匹配。
- ARP 录制会同步写入手脚 IK/FK 开关关键帧，避免停止接收后恢复 IK 模式导致已录制的 FK 腿部曲线不参与回放。
- 录制帧号按真实录制时间和 Blender 工程 FPS 换算，避免接收采样率与工程 FPS 不一致导致回放快放或慢放。
- 停止录制重新绑定 Action 时会同步绑定 Blender 5.1 action slot，避免 layered action 有数据但不参与回放。
- Shape Key 录制会写入 Shape Keys 数据块对应的 `KEY` action slot，避免误写到对象 slot 后无法回放。
- 录制 tick 跨过多个工程帧时会按前后采样插值补齐中间帧，避免保存后的 Action 出现跳帧和台阶型关键帧。
- 录制补帧已提升为样本级插值：位置与 Shape Key 线性补帧，骨骼旋转按 quaternion slerp 补帧后再写回原旋转模式，避免 Euler 分量补帧带来的局部不均匀。
- 录制烘焙后的关键帧统一写为 `LINEAR` 插值，避免默认 `BEZIER` 缓动继续制造不均匀曲线段。
- 每个录制采样会把目标层当前骨骼姿态缓存到内存轨道，停止录制时再写入 Action。
- 每个录制采样会把当前目标 Shape Key 值缓存到内存轨道，停止录制时再写入 Shape Key Action。
- 录制帧号由插件会话管理，并按真实录制时间映射到 Blender 工程帧号。
- 停止录制或停止接收后会重新绑定录制 Action，不删除已写入数据。
- 已新增 `runtime/target_runtime.py`，开始抽离统一目标求值链路。
- 目标角色实时预览与录制采样已开始共用 `TargetEvaluatedSample`，不再分别直接读取当前对象状态。
- 接收层已开始维护 `RawMotionFrame` 队列，目标预览与录制可按原始帧时间戳消费队列结果。
- 实时目标预览当前已改为只消费最新一帧原始输入，录制仍保留完整原始帧序列离线烘焙，避免主线程为追赶积压队列重放整批历史帧。
- 预览骨架和通用目标骨架当前已在接收上下文内预编译 runtime entry，减少主线程每帧骨名查找和 `pose[...]` 解析。
- 预览层和目标层当前已在接收上下文内预计算 bone rest quaternion / inverse quaternion，并拆分 VMC 的位置 / 旋转转换，减少主线程每帧四元数反求和无用对象分配。
- `TargetEvaluatedSample` 当前已在实时链路内直接保留 `Vector / Quaternion`，只在录制缓存阶段再做纯数值化，减少主线程实时路径的 tuple <-> mathutils 来回转换。
- 停止录制后的保存流程已拆成“准备保存 -> 重建采样 -> 写入动作曲线”三段执行，Blender 状态栏会持续显示进度；录制前状态恢复、视图层刷新和目标上下文构建也已纳入分段流程，减少进度条出现前的同步卡顿。
- 接收线程当前会同步维护 canonical 骨骼缓冲，主线程实时驱动不再每 tick 对当前骨骼缓冲整表规范化。
- 目标骨骼与 Shape Key 的实时应用阶段当前已开始直接复用 sample 内的 `pose_bone / key_block` 引用，减少主线程按名称重复回查 RNA 集合。
- 主线程在存在原始帧队列时当前不再先额外复制一次当前缓冲快照，避免“先复制再丢弃”的无效分配。
- ARP 目标链路当前会在接收上下文内预编译 source 分组入口、IK/FK 控制骨列表和约束依赖映射，减少主线程每帧重建字典以及重复扫描约束列表。
- 目标对象起始世界分解、世界逆旋转，以及 ARP 目标骨起始平移/缩放当前会缓存到接收上下文里，减少主线程每帧 `decompose()`、`to_translation()`、`to_scale()` 和 `to_3x3().inverted()`。
- canonical 骨骼输入当前会在运行时求值里优先走直接 key 命中，避免 `get_vmc_bone_pose()` 在主链上重复构建大小写/规范化查找表。
- `TargetEvaluatedSample` 的对象和骨骼项当前不再冗余保存 `rotation_mode` 字符串，应用阶段也直接复用已有 `Vector / Quaternion`，减少主线程字符串分配和 mathutils 复制。
- 目标层实时应用当前已去掉未被录制链路消费的 `driven_bones / driven_shapes` 集合构建，以及 `record_current_sample()` 的历史无效参数，减少主线程每帧临时分配。
- ARP 与预览路径当前会在接收上下文里缓存骨骼 `Bone` 引用、`matrix_local` 和父骨本地矩阵，减少实时求值阶段反复读取静态 RNA 数据与父链本地矩阵。
- `runtime/driver.py` 中已退出实时主链的旧 ARP 直驱实现和重复 helper 当前已清理，避免双实现并存继续干扰性能分析与后续维护。
- 预览骨架对象位移、对象旋转、骨骼旋转和眼球旋转当前也已改成按分量直接比较 `mathutils` 值，只有真的写入 Blender RNA 时才构造缓存 tuple；同时 `dirty_bone_names` 已改成在主线程入口统一构造一次并复用给预览层和目标层，继续减少实时热路径的临时对象分配。
- 预览层和目标层实时应用当前都会返回“本帧是否真的写入了 Blender RNA”；`apply_timer()` 已改成只在发生真实预览/目标写入，或录制状态计数需要刷新 UI 时才触发 `VIEW_3D` redraw，减少高频输入下无效的侧栏与视口重绘。
- `VIEW_3D` redraw 当前已缓存窗口布局中的目标 area 列表，只在 window/screen 结构变化或缓存失效时才重建，避免每次 redraw 都重新扫描所有 window 和 area。
- 目标骨架求值当前已在接收上下文里预编译 `source -> runtime entry / group index` 查找表；少量 `dirty_bone_names` 输入时，Generic / MMD 直接命中对应 source entry，ARP 只进入受影响的 runtime group，减少每帧按整套入口扫描后再逐条跳过的 Python 循环开销。
- VRM 预览骨架当前也已在预览上下文里预编译 `source -> runtime entry` 查找表；少量 `dirty_bone_names` 输入时，预览层直接命中对应 source entry，不再每帧先扫描整套中间骨架入口再逐条跳过。
- ARP 目标骨架当前也已从“按受影响 group 进组后再判脏 source”收紧为“按 `ARP_RUNTIME_SOURCE_ORDER` 直接命中受影响 runtime entry 顺序执行”，继续减少 group 构建、组内 membership 判断和嵌套 Python 循环。
- ARP runtime entry 当前还已预编译“是否有父骨”和“是否需要生成 effective pose matrix”两个布尔位，实时求值最内层循环不再每骨重复做 `parent is None` 与 `target_name in required_pose_targets` 判断。
- 运行时上下文当前已移除仅构建阶段使用的 `arp_required_pose_targets` 死字段；预览层与 Generic 目标层在按 `dirty_bone_names` 直达 runtime entry 后，也已去掉循环内重复的二次 membership 判断。
- ARP 约束辅助链路当前会在求值入口一次取出 `copy_transforms / child_of / copy_location` 三类约束映射，并把 dict 引用直接沿递归与单骨流程传递，减少 `_calculate_arp_target_pose_matrix()` 及其辅助函数里的重复 `context.get(...)` 查表。
- ARP 单骨求值当前会在 entry 循环开始先取一次 `raw_pose`，局部旋转与世界旋转两条分支直接复用；并已移除 `_source_local_rotation_from_raw()` 与按骨名回查的 `_source_world_rotation_from_raw()`，减少 helper 往返和重复输入查表。
- Generic / MMD 目标骨架求值当前已去掉每帧对 runtime entry 的额外 `tuple(...)` 包装；同时未使用的 `_remap_local_delta_between_rest_axes()` 旧 helper 已移除，继续清理热路径中的无效对象构造与死代码。
- `apply_timer()` 当前在直接消费实时缓冲、未走 `raw_frame_queue` 时，已不再把三类 dirty set 额外复制成 tuple；而是直接接管旧 set 引用后再重置全局缓冲，减少主线程每帧的无意义容器复制。
- 接收器空闲、没有任何新数据和脏状态时，`apply_timer()` 当前已不再默认按约 10Hz 持续触发 3D 视图 redraw；现在只在存在最近数据包、预览面板年龄文案可能变化时才做约 1Hz 低频刷新。
- `TargetEvaluatedSample` 当前已改为按需填充通道：实时求值阶段不再默认复制对象 / 骨骼当前位置和 basis 旋转，录制转换阶段再补齐默认值，减少主线程每帧 `location.copy()` 与 `matrix_basis.to_quaternion()`。
- ARP 实时求值当前只为后续父链 / 约束依赖需要的目标骨生成完整 pose matrix，叶子骨不再无差别执行第二次 `convert_local_to_pose()` 和 `Matrix.LocRotScale()`。
- 录制转换阶段当前已去掉 `dict.get(default)` 里会提前求值的 fallback 四元数生成，减少离线重建与停止录制保存阶段的无效 mathutils 开销。
- 录制阶段当前会在开始录制时缓存目标对象和目标骨骼的 `rotation_mode`，离线重建与保存时不再每帧回查接收前 snapshot。
- 预览骨架和预览骨骼的 `rotation_mode = QUATERNION` 当前只会在接收上下文初始化后设置一次，不再在实时预览每帧重复写同一个 RNA 属性。
- 仅使用中间层预览、未绑定任何目标对象时，主线程当前会在预览更新后直接早退，不再继续创建和应用空的目标 sample。
- 预览面部与目标面部当前会缓存 source key 到 `KeyBlock` 的直接引用，减少实时链路每帧 `key_blocks.get(name)` 和重复集合查询。
- 开始录制后当前会缓存目标骨骼 `PoseBone` 引用和目标 Shape Key `KeyBlock` 引用，离线重建与初始采样不再反复 `pose.get()` / `key_blocks.get()`。

仍需验证：

- 真实 VMC 身体数据长时间录制回放。
- 真实 ARKit / VMC 表情数据长时间录制回放。
- 骨骼与形态键同时录制时的性能。

- 多段连续录制、清帧、重新录制在同一 Action 上的可编辑性。

状态：`持续进行`

目标：

验证清单：

- ARP 角色长时间实时预览。
- MMD 角色长时间实时预览。
- 骨骼和形态键同时驱动。

- 更清晰的错误报告和缺失项定位。

1. 用 Blender MCP 读取当前工程 MMD 骨架，建立开发测试基准。
