# Development Plan

## 项目阶段判断

VMC Link 当前已经完成基础链路建设，进入“生产适配前调试与模块拆分”阶段。

当前不再把重点放在新增更多输入源，而是围绕以下链路继续收敛：

```text
VMC / RhyLive ARKit 输入
  -> VRM / ARKit 中间层实时预览
  -> 可切换目标映射模块
  -> 目标骨架 / 目标面部实时驱动
  -> Blender 动作与形态键录制
```

## 当前开发状态

### 已完成并验证

#### 1. 基础接收与预览链路

- VMC UDP 接收可用。
- RhyLive ARKit `/Face` 接收可用。
- OSC 收包与解码已放到后台线程。
- Blender 对象、骨骼和 Shape Key 写入保留在主线程。
- VRM 预览骨架可实时驱动。
- ARKit 调试面部可实时驱动。
- 数据预览已移到底部并默认折叠。

#### 2. 中间层与目标层解耦

- 预览层对象：
  - `vmc_link_preview_armature`
  - `vmc_link_preview_face_object`
- 目标层对象：
  - `vmc_link_armature`
  - `vmc_link_face_object`
- 预览层只看原始数据，不读取用户映射。
- 目标层按映射驱动，并受 `实时驱动目标对象` 开关控制。
- 停止接收时可以恢复接收开始前的骨架与面部状态。

#### 3. VRM 中间层

- `VMC_TEST_DUMMY` 预览骨架创建 / 重建可用。
- 创建和重建限定在 `VMC_Link` 集合内，避免误删场景其他骨架。
- 支持按目标骨架执行长度校准。
- 当前校准逻辑已覆盖：
  - 躯干整体比例。
  - 四肢主链长度。
  - 手掌长度。
  - 手指分节长度。
  - 校准后对象位置回归和骨盆点对齐。

#### 4. ARKit 调试面部

- 从 `assets/Arkitface.blend` 只追加 `ArkitFace` 集合。
- 导入对象放入插件专用集合 `VMC_Link`。
- 不覆盖目标面部槽位。

#### 5. Auto Rig Pro 骨骼映射

ARP 映射模块当前已验证为可用阶段，仅待生产环境继续具体调优。

已完成能力：

- 标准 ARP 控制骨检测。
- 标准 ARP 映射自动填充。
- 当前映射校验。
- 内建 `arp_fk_humanoid.json` 预设。
- 接收时自动切换四肢 FK，停止后恢复。
- VRM T Pose 与 ARP A Pose 姿态差校准。
- 手指局部 basis delta 驱动。
- 拇指 rest 轴转换，避免内弯变外翻。
- 脚和脚趾使用相对起始姿态的 delta 旋转。
- ARP 根运动只写 `c_traj.location`，保持 `rig` 对象和 `c_pos` 不动。
- 运行时缓存骨引用、映射和校准结果。
- 跳过未变化骨骼，减少不必要写入。

当前结论：

- ARP 路径已经从“实验实现”进入“可用并待生产调优”。
- 后续不应继续把 ARP 逻辑堆进通用映射 UI，而应拆成可切换目标骨架类型的独立模块。

#### 6. MMD 形态键映射

当前已新增并验证：

- VMC -> MMD Shape Key 映射表。
- ARKit 52 Key -> MMD Shape Key 映射表。
- 两套映射在实时预览状态下功能可用。

现有预设：

- `presets/vmc_blend_maps/arona_vmc.json`
- `presets/arkit_blend_maps/arona_arkit.json`

方向判断：

- `VMC -> 形态键` 继续保留为兼容 / 备选方案。
- 后续面部主线仍以 `ARKit -> MMD` 为主。

### 当前主要问题

#### 1. 录制功能需要真实接收流验证

当前已完成目标层关键帧写入的基础修复，但还需要用真实 VMC / ARKit 接收流进行长时间回放验证。

已完成：

- 开始录制时显式准备目标骨架 Action 和目标面部 Shape Key Action。
- 录制入口已提升为独立录制面板，主面板不再混放录制按钮。
- 支持显式录制开始帧 / 结束帧，并在达到结束帧后自动停止。
- 支持按当前录制目标通道清空指定范围内关键帧。
- 支持可配置的起始姿态过渡：从接收会话启动时保存的目标初始姿态过渡到过渡结束帧对应的有效动捕姿态。
- 录制会复用当前 Action；没有 Action 时才创建。
- 录制期间会临时从目标对象解绑 Action，只缓存目标层结果，停止录制时统一烘焙 fcurve，避免 Action 评估把 ARP 控制骨覆盖回 A Pose。
- 录制会按接收开始前的骨骼 `rotation_mode` 写入旋转曲线，避免运行时 quaternion 模式与回放时 XYZ 模式不匹配。
- ARP 录制会同步写入手脚 IK/FK 开关关键帧，避免停止接收后恢复 IK 模式导致已录制的 FK 腿部曲线不参与回放。
- 录制帧号按真实录制时间和 Blender 工程 FPS 换算，避免接收采样率与工程 FPS 不一致导致回放快放或慢放。
- 停止录制重新绑定 Action 时会同步绑定 Blender 5.1 action slot，避免 layered action 有数据但不参与回放。
- Shape Key 录制会写入 Shape Keys 数据块对应的 `KEY` action slot，避免误写到对象 slot 后无法回放。
- 录制 tick 跨过多个工程帧时会按前后采样插值补齐中间帧，避免保存后的 Action 出现跳帧和台阶型关键帧。
- 录制补帧已提升为样本级插值：位置与 Shape Key 线性补帧，骨骼旋转按 quaternion slerp 补帧后再写回原旋转模式，避免 Euler 分量补帧带来的局部不均匀。
- 录制烘焙后的关键帧统一写为 `LINEAR` 插值，避免默认 `BEZIER` 缓动继续制造不均匀曲线段。
- 每个录制采样会把目标层当前骨骼姿态缓存到内存轨道，停止录制时再写入 Action。
- 每个录制采样会把当前目标 Shape Key 值缓存到内存轨道，停止录制时再写入 Shape Key Action。
- 录制帧号由插件会话管理，并按真实录制时间映射到 Blender 工程帧号。
- 停止录制或停止接收后会重新绑定录制 Action，不删除已写入数据。

仍需验证：

- 真实 VMC 身体数据长时间录制回放。
- 真实 ARKit / VMC 表情数据长时间录制回放。
- 骨骼与形态键同时录制时的性能。
- 暂停 / 继续后继续录制的时间轴连续性。

#### 2. 映射模块仍未完全解耦

当前骨骼映射 UI 已经被 ARP 辅助区占据，后续加入 MMD 骨骼映射时会变得混乱。

需要新增“目标骨架类型”概念：

- 通用骨架
- Auto Rig Pro
- MMD

目标骨架类型应影响：

- 可见的辅助 UI。
- 自动检测逻辑。
- 自动填充逻辑。
- 校验报告。
- 内建预设入口。
- 运行时映射策略。

#### 3. 缺少 VRM -> MMD 骨骼映射模块

当前已有 MMD 形态键映射，但骨骼侧还没有 MMD 标准骨骼模块。

需要新增：

- MMD 标准骨骼检测。
- VRM -> MMD 默认骨骼映射。
- MMD 映射自动填充。
- MMD 映射校验。
- 内建 MMD 骨骼预设。
- 与 ARP 并列的 UI 辅助区。

#### 4. 生产环境验证不足

当前多数验证来自开发场景和单个角色。

后续需要建立生产验证清单：

- 不同 ARP 角色。
- 至少一套标准 MMD 角色。
- 长时间实时预览。
- 录制回放。
- 表情和骨骼同时录制。
- 固定中心开关切换。
- 暂停 / 继续 / 停止恢复。
- 预设保存、加载、重建缓存。

## 下一阶段路线

## Phase 1：动作与形态键录制修复

状态：`基础修复完成，待真实接收流验证`

目标：

- 把录制功能修到可实际用于 Blender 工程，并通过真实接收流回放验证。

范围：

- 骨骼关键帧录制。
- Shape Key 关键帧录制。
- 录制状态管理。
- 停止接收与停止录制的边界。
- 录制结果回放验证。

建议实现顺序：

1. 已完成：梳理当前录制代码路径，明确哪些对象和骨骼会被 keyframe。
2. 已完成：拆分“实时驱动状态恢复”和“录制结果保留”。
3. 已完成：录制目标限定为目标层骨架和目标面部。
4. 已完成：补 Shape Key 录制，确保 VMC / ARKit 两种来源都能写入动画轨道。
5. 已完成：修复录制 Action 评估覆盖实时 ARP 控制骨，导致录制结果保持 A Pose 的问题。
6. 已完成：修复运行时 quaternion 录制曲线与回放时 XYZ 骨骼模式不匹配的问题。
7. 已完成：修复 Blender 5.1 layered action slot 未绑定导致 fcurve 数据不参与回放的问题。
8. 已完成：用 Blender MCP 验证非 `c_traj` ARP 控制骨可写入并回放两帧不同旋转值。
9. 已完成：修复 ARP 下半身录制后回放不动的问题，录制 Action 会写入手脚 IK/FK 开关以保持 FK 曲线生效。
10. 已完成：修复录制动作快放问题，录制帧号改为按真实时间和工程 FPS 计算。
11. 已完成：修复 Shape Key 录制写入错误 action slot，导致曲线存在但回放不生效的问题。
12. 已完成：修复录制 tick 跨帧时只写当前帧导致关键帧跳帧的问题。
13. 已完成：将录制阶段改为内存缓存、停止录制时统一烘焙 Action，降低录制越久越卡的问题。
14. 已完成：修复补帧直接复用同一末端姿态导致关键帧数值不均匀的问题，改为按相邻采样插值补帧。
15. 已完成：修复录制烘焙默认 `BEZIER` 插值导致曲线段缓动不均匀的问题，统一改为 `LINEAR`。
16. 已完成：将旋转补帧从 Euler 分量线性插值改为 quaternion slerp，再回写目标旋转模式，减少多轴旋转关键帧的不均匀问题。
17. 待完成：在真实 VMC / ARKit 接收流下验证长时间录制结果。
18. 待开发：添加 ARP 腿部 FK 动作转 IK 控制器工具，用于后期脚部 IK 手动 K 帧修复。

验收标准：

- 开始录制后，目标骨架动作被写入 Action。
- 目标面部 Shape Key 被写入关键帧。
- 停止录制后时间轴可回放。
- 停止接收会恢复实时驱动前状态，但不破坏已录制 Action。
- 录制过程中视口仍能保持可接受帧率。
- 真实接收流下连续录制不会出现丢关键帧、断帧或录制状态悬挂。

## Phase 2：映射模块解耦与目标骨架类型切换

状态：`待开始`

目标：

- 不再让 ARP 辅助逻辑直接耦合到通用骨骼映射面板。
- 为 MMD 骨骼映射接入预留清晰入口。

建议新增 Scene 属性：

- `vmc_link_target_rig_type`
  - `GENERIC`
  - `ARP`
  - `MMD`

建议模块结构：

```text
mapping/
  mapper.py          # 通用映射与缓存
  preset_store.py    # JSON 预设
  arp.py             # ARP 目标骨架模块
  mmd.py             # MMD 目标骨架模块
```

UI 调整：

- 骨骼映射面板顶部增加“目标骨架类型”菜单。
- 选择 `Auto Rig Pro` 时显示 ARP 检测、自动填充、校验。
- 选择 `MMD` 时显示 MMD 检测、自动填充、校验。
- 选择 `通用骨架` 时只显示通用映射和预设。

运行时约束：

- `runtime/driver.py` 不应直接包含大量 ARP / MMD UI 或检测逻辑。
- 专用模块负责生成映射和校验。
- 运行时只消费最终映射结果和必要的目标类型策略。

验收标准：

- 切换目标骨架类型不会破坏现有 ARP 实时驱动。
- ARP 辅助区只在 ARP 类型下显示。
- MMD 类型可以独立显示自己的检测与填充入口。
- 现有 JSON 预设仍能加载、保存和刷新。

## Phase 3：VRM -> MMD 骨骼映射模块

状态：`待开始`

目标：

- 新增 MMD 标准骨骼目标适配能力。

建议范围：

- 定义 MMD 标准骨骼名表。
- 定义 VRM 中间层骨骼到 MMD 骨骼的默认映射。
- 支持左右骨、手指、眼睛、上半身链路。
- 提供 MMD 骨架检测。
- 提供自动填充 MMD 映射。
- 提供 MMD 映射校验。
- 新增内建预设，例如：
  - `presets/bone_maps/mmd_humanoid.json`

需要重点验证：

- MMD 骨架坐标轴与 VRM 中间层轴向差异。
- 手指旋转方向。
- 脚和脚趾旋转。
- 根骨 / センター / 下半身 / 上半身 的职责划分。
- 是否需要类似 ARP 的目标专用根运动策略。

验收标准：

- 标准 MMD 骨架可被识别。
- 自动填充不会覆盖用户已有手动映射。
- 校验报告能区分必需骨、可选骨、手指、眼睛。
- 实时驱动时不会影响 ARP 路径。
- 可保存和加载 MMD 骨骼预设。

## Phase 4：生产环境验证与优化

状态：`持续进行`

目标：

- 用真实角色和真实动捕数据验证插件可用性。

验证清单：

- ARP 角色长时间实时预览。
- MMD 角色长时间实时预览。
- 骨骼和形态键同时驱动。
- 骨骼和形态键同时录制。
- 暂停、继续、停止、恢复。
- 固定中心开启 / 关闭。
- 预设保存、加载、刷新。
- 数据预览展开和折叠状态下的性能。

优化方向：

- 目标层独立驱动频率上限。
- Shape Key 写入节流。
- 录制时的关键帧降采样。
- 更清晰的错误报告和缺失项定位。
- 面向生产角色的快速验证按钮。

## 当前优先级

1. 用真实 VMC / ARKit 接收流验证录制回放，继续修复录制边界问题。
2. 拆分映射模块，加入目标骨架类型切换菜单。
3. 接入 VRM -> MMD 骨骼映射模块。
4. 做生产环境验证，基于真实问题继续 debug 和性能优化。

## 当前边界

- 不新增新的输入协议，除非当前 VMC / ARKit 链路已经完成生产验证。
- 不把 ARP 和 MMD 的特殊逻辑混进通用映射路径。
- 不在后台线程写 Blender RNA。
- 不引入会掩盖问题的宽松兜底映射。
- VMC 表情映射保留，但后续面部主线是 ARKit -> MMD。
